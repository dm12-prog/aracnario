# Aracnário 🕷️

An offline-first field guide to **every spider recorded in Portugal** — over 1,100 species across 57 families, each with a painterly, family-accurate illustration. Browse, read, and log the ones you meet; your sightings, favourites and vivarium are remembered between visits.

**[Open the app →](https://dm12-prog.github.io/aracnario/)** *(enable GitHub Pages to serve it)*

## Highlights

- **1,129 species, fully offline.** The whole catalogue is embedded in `catalogue.js` and loaded with a plain `<script>` tag — no runtime `fetch()`, so the app works installed, offline, and even opened straight from `file://`. Photos are the only thing that needs a connection.
- **Painterly procedural sprites.** Every species gets a soft, naturalist-plate illustration generated from its family's real body plan — leg stance, abdomen shape, and eye arrangement — with a colour/pattern seeded from its iNaturalist id. No two look alike; each still reads as its family. See `src/sprites.js` + `src/adapt.js`.
- **A collectible field guide.** 45 species are hand-authored with lore, medical notes, size and phenology; the rest are ranked into a rarity pyramid by how often they're actually recorded in Portugal. Log sightings, earn ranks and achievements, favourite species into a little **vivarium** where they wander and can be fed.
- **Installable PWA.** Add to home screen; a service worker (`sw.js`) caches the shell for offline use.

## Data sources

- **GBIF** occurrence records (Portugal) + **World Spider Catalog** — the species backbone (`data/layer1_skeleton.json`).
- **iNaturalist** API — common names, descriptions, and CC-licensed photos with attribution (`data/species_enrichment.json`).
- Hand-authored notes for the 45 featured species.

All illustrations are stylised, family-based reconstructions — **a learning aid, not an identification key or a medical guide.**

## Build

The embedded catalogue is generated by merging the data sources:

```bash
python3 data/build_catalogue.py   # -> catalogue.js
```

## Run locally

Because everything is embedded, you can just open `index.html`. To exercise the service worker / PWA, serve over HTTP:

```bash
python3 -m http.server 8137
# then open http://localhost:8137/index.html
```

## Project layout

```
index.html            the app (shell + catalogue/vivarium UI)
catalogue.js          generated — the full embedded species dataset
src/adapt.js          family morphology + per-species art descriptor
src/sprites.js        painterly SVG sprite renderer
src/db.js             tiny IndexedDB key/value store (logs, favourites)
sw.js, manifest.json  PWA offline shell
data/                 source datasets + build scripts
```
